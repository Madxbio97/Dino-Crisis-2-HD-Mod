# Dino Crisis 2 — DX9 Texture Dumper & Replacer

ASI-плагин для дампа и замены **всех** текстур в Dino Crisis 2 (Sourcenext) с патчем Classic Rebirth.

Перехватывает DX9-вызовы `SetTexture` и `EndScene` через detour-хук на уровне `d3d9.dll`, что обеспечивает совместимость с любой версией Classic Rebirth.

## Возможности

- **Дамп** всех текстур игры в формате WebP (фоны, маски, спрайты, 3D-текстуры, UI)
- **Замена** текстур из папки `hires\textures\` — поддерживается произвольное разрешение
- **Автодетект смены комнат** — двухуровневый кэш с быстрым хешем предотвращает наложение текстур из предыдущих сцен
- **Корректная альфа** — фоны (320×240) сохраняются непрозрачными, маски и спрайты сохраняют прозрачность

## Требования

- **Dino Crisis 2** (Sourcenext, японское издание)
- **Classic Rebirth** (`ddraw.dll`) — DX9-обёртка для игры
- **ASI Loader** (`dinput.dll`) — например, Ultimate ASI Loader
- **libwebp.dll** — библиотека кодирования/декодирования WebP (из мода TeamX или скачать отдельно)

## Установка

1. Убедитесь, что Classic Rebirth (`ddraw.dll`) установлен и игра запускается
2. Поместите в папку с `Dino2.exe`:
   - `dino2hd_ext.asi`
   - `dinput.dll` (ASI Loader)
   - `libwebp.dll`
3. Запустите игру

## Структура файлов

```
Dino2.exe
ddraw.dll              ← Classic Rebirth
dinput.dll             ← ASI Loader
dino2hd_ext.asi        ← этот плагин
libwebp.dll            ← кодек WebP
dino2hd_ext.log        ← лог (создаётся автоматически)
dump/
  textures/
    A3F7B210_320x240.webp   ← дампы текстур
    ...
hires/
  textures/
    A3F7B210_320x240.webp   ← замены (то же имя файла)
    ...
```

## Использование

### Дамп текстур

1. Запустите игру с установленным плагином
2. Загрузите сохранение, походите по локациям
3. Текстуры автоматически сохраняются в `dump\textures\`
4. Имя файла: `<HASH>_<W>x<H>.webp`, где HASH — FNV-1a хеш содержимого текстуры

### Замена текстур

1. Скопируйте нужный файл из `dump\textures\` в `hires\textures\` (имя файла должно совпадать)
2. Отредактируйте файл в графическом редакторе (GIMP, Photoshop и т.д.)
3. Сохраните как WebP (lossless рекомендуется)
4. Перезапустите игру — замена применится автоматически

> **Размер замены может отличаться от оригинала.** Плагин создаёт новую DX9-текстуру нужного размера. Например, оригинальный фон 320×240 можно заменить на 1280×960.

### Конфигурация

Режимы задаются в исходном коде (перекомпиляция):

```c
static int g_dump    = 1;  // 1 = дампить текстуры
static int g_replace = 1;  // 1 = заменять из hires\textures\
```

## Сборка из исходников

### MSVC (Visual Studio)

Откройте **x86 Native Tools Command Prompt**:

```cmd
cl /LD /O2 dino2hd_ext.c /link /OUT:dino2hd_ext.asi kernel32.lib user32.lib
```

### MinGW (MSYS2)

В терминале **MSYS2 MINGW32**:

```bash
pacman -S mingw-w64-i686-gcc   # один раз
i686-w64-mingw32-gcc -shared -O2 -o dino2hd_ext.asi dino2hd_ext.c -lkernel32 -luser32
```

> **Важно:** только 32-bit (x86). Игра 32-битная.

|--------|-----|---------------|
| 320×240 | Пререндеренный фон | Непрозрачный (alpha = 255) |

## Как это работает

1. При загрузке ASI создаёт временный DX9-девайс для получения адресов функций `EndScene` и `SetTexture` в `d3d9.dll`
2. Устанавливает **detour-хуки** (перезапись первых байт функции на `JMP`) — это перехватывает вызовы через любой девайс, включая созданный Classic Rebirth
3. При каждом `SetTexture`:
   - Вычисляется **быстрый хеш** (3 строки пикселей) для детекта изменений
   - Если контент изменился — пересчитывается **полный хеш** и ищется замена
   - Результат кэшируется на 30 кадров
4. Замены хранятся в отдельном кэше `хеш → DX9-текстура` и переиспользуются

## Устранение неполадок

- **Игра не запускается:** проверьте, что `dinput.dll` — это ASI Loader, а не оригинальный файл DirectInput
- **Лог пустой:** ASI Loader не загружает плагин. Убедитесь, что файл имеет расширение `.asi`
- **`frames=0` в логе:** Classic Rebirth не создал DX9-девайс. Проверьте, что `ddraw.dll` — это Classic Rebirth
- **Текстуры не дампятся:** проверьте наличие `libwebp.dll` в папке с игрой
- **Замена не применяется:** имя файла в `hires\textures\` должно **точно** совпадать с именем в `dump\textures\`
