# Dino Crisis 2 — HD Mask Generator

Automatically generates pixel-perfect high-resolution mask textures from upscaled backgrounds for Dino Crisis 2.

## Background

Dino Crisis 2 renders scenes using three layers:

1. **Background** (320×240) — a pre-rendered 2D image filling the entire screen.
2. **3D models** — the player character and enemies, rendered on top of the background.
3. **Masks** (512×512) — fragments of the background drawn on top of the 3D models to create depth (e.g., a fence that the character walks behind).

Each mask texture contains tiles copied from its corresponding background. The game draws these tiles at specific screen positions using `DrawPrimitiveUP` with UV coordinates that map regions of the 512×512 mask to screen rectangles overlapping the background.

When the background is replaced with an AI-upscaled HD version (e.g., 1280×960), the original 512×512 mask no longer matches — its pixels are from the old low-resolution background. Simply upscaling the mask with bicubic/AI doesn't work because the RGB values must be **identical** to the HD background pixels underneath, or visible seams appear.

## How It Works

This script solves the problem by using coordinate data captured by the diagnostic plugin (`dc2_diag.asi`):

1. **Read tile mappings** from `dc2_pairs.tsv` — each row describes one tile: which mask texture, which background, the UV region within the mask, and the destination rectangle on screen.

2. **For each HD mask pixel**, trace backwards through the coordinate chain:
   - HD mask pixel → original mask texel position (divide by scale factor)
   - Original mask texel → UV position within the tile
   - UV position → normalized position (0..1) within the tile
   - Normalized position → screen position (using destination rectangle)
   - Screen position → HD background pixel (using screen-to-background ratio)

3. **Copy the RGB** from the HD background at the computed position. This guarantees pixel-perfect alignment — the mask pixels are literally cut from the same HD background image.

4. **Smooth the alpha channel** using bicubic interpolation. The original mask has binary alpha (0 or 255) at 512×512. When upscaled to 2048×2048, nearest-neighbor would produce jagged edges. Bicubic interpolation creates smooth gradients (0–255) at the boundaries, producing anti-aliased edges.

## Requirements

- Python 3.8+
- Pillow: `pip install Pillow`
- NumPy (installed automatically with Pillow)

## Input Files

| File | Source | Description |
|------|--------|-------------|
| `dc2_pairs.tsv` | Generated by `dc2_diag.asi` | Tab-separated tile coordinate mappings |
| `dump/textures/*.webp` | Generated by `dino2hd_ext.asi` | Original texture dumps (used for mask alpha channel) |
| `hires/textures/*.webp` | User-provided (AI upscaled) | HD background images (source for mask RGB pixels) |
| `mask_overrides.tsv` | User-created (optional) | Manual corrections for misidentified pairs |

### File Naming Convention

All texture files follow the pattern `{HASH}_{WIDTH}x{HEIGHT}.webp`:

- `BA06487B_320x240.webp` — original background (hash BA06487B, 320×240)
- `F9F692E7_512x512.webp` — original mask (hash F9F692E7, 512×512)

HD replacements keep the **same filename** as the original but contain higher-resolution image data. For example, `BA06487B_320x240.webp` in `hires/textures/` might actually be a 1280×960 image.

## Usage

### Generate All Masks

```bash
python make_hd_masks.py
```

Uses default paths: `dc2_pairs.tsv`, `dump/textures`, `hires/textures`.

### Test a Single Background

```bash
python make_hd_masks.py --test BA06487B
```

Generates only the mask for background BA06487B. Useful for debugging.

### Custom Paths

```bash
python make_hd_masks.py --tsv dc2_pairs.tsv --dump dump/textures --hires hires/textures --out hires/textures
```

### All Options

| Option | Default | Description |
|--------|---------|-------------|
| `--tsv` | `dc2_pairs.tsv` | Path to the tile coordinate mappings file |
| `--dump` | `dump/textures` | Folder with original texture dumps |
| `--hires` | `hires/textures` | Folder with HD upscaled backgrounds |
| `--out` | `hires/textures` | Output folder for generated HD masks |
| `--overrides` | `mask_overrides.tsv` | Manual background→mask corrections |
| `--test` | *(none)* | Process only one background hash |

## Output

Generated HD masks are saved to the output folder with the original mask filename:

```
hires/textures/F9F692E7_512x512.webp    ← 2048×2048 HD mask (lossless WebP)
```

The mask is ready for the texture replacement plugin (`dino2hd_ext.asi`) to load at runtime.

## Manual Overrides

During room transitions, the diagnostic plugin may occasionally associate a mask with the wrong background. When this happens, the generated mask will show incorrect pixels (from a different room's background).

To fix this, create `mask_overrides.tsv`:

```
# bg_hash	correct_mask_hash
526B109C	413FEA13
A11E80A0	4DEA6744
```

Tab-separated, one pair per line, `#` for comments. Overrides take priority over auto-detected pairs.

## TSV Format

The `dc2_pairs.tsv` file contains one row per mask tile with the following columns:

| Column | Type | Description |
|--------|------|-------------|
| `bg_hash` | hex string | FNV-1a hash of the background texture content |
| `overlay_hash` | hex string | FNV-1a hash of the mask texture content |
| `overlay_w` | int | Mask texture width (always 512) |
| `overlay_h` | int | Mask texture height (always 512) |
| `draw_order` | int | Draw call index within the frame |
| `dst_x0` | float | Screen destination left edge |
| `dst_y0` | float | Screen destination top edge |
| `dst_x1` | float | Screen destination right edge |
| `dst_y1` | float | Screen destination bottom edge |
| `uv_u0` | float | Mask UV left (0.0–1.0) |
| `uv_v0` | float | Mask UV top (0.0–1.0) |
| `uv_u1` | float | Mask UV right (0.0–1.0) |
| `uv_v1` | float | Mask UV bottom (0.0–1.0) |

## Screen Resolution

The script auto-detects the screen resolution from the maximum destination coordinates in the TSV. All data in a single TSV must be collected at the **same resolution** (recommended: 1280×960). Mixing data from different resolutions will produce incorrect masks.

## Algorithm Details

### Pair Detection

A mask is associated with a background if:
- The mask texture is 512×512
- The mask appears exclusively with one background (not shared across rooms)

Masks that appear with multiple backgrounds are ambiguous and must be resolved via `mask_overrides.tsv`.

### Pull-Based Rendering

The script uses a **pull approach**: for each pixel in the output HD mask, it computes where that pixel should come from in the HD background. This is the reverse of the game's rendering (which pushes tiles from mask to screen). The pull approach ensures every output pixel is filled exactly once with no gaps.

### UV Padding

Adjacent tiles in the original mask share edges. At HD resolution, floating-point rounding can leave 1-pixel gaps between tiles. The script expands each tile's UV region by ±1 texel to ensure complete coverage.

### Alpha Anti-Aliasing

The original mask alpha channel is binary: 0 (transparent) or 255 (opaque). At 4× upscale, this produces blocky staircase edges. The script upscales the alpha channel separately using **bicubic interpolation** (`Image.BICUBIC`), producing smooth gradients at boundaries. The RGB pixels still come from the HD background (pixel-perfect), only the alpha is interpolated.

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| Wrong pixels in mask | Mask paired with wrong background | Add correct pair to `mask_overrides.tsv` |
| Gaps between tiles | Resolution mismatch in TSV data | Re-collect all data at the same resolution |
| `SKIP: mask not in dump` | Original mask texture not dumped | Re-run game with dump enabled to capture it |
| `SKIP: HD bg not in hires` | HD background not provided | Upscale the background and place in `hires/textures/` |
| Jagged mask edges | Using old version without alpha smoothing | Update to latest `make_hd_masks.py` with bicubic alpha |
| `BG not found` with `--test` | Hash not in TSV or wrong case | Check hash exists in `dc2_pairs.tsv` |
